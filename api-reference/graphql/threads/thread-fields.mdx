title: 'Thread fields'
---

Thread fields allow you to extend Plain's thread data model. The thread fields which you want to support have to confirm with a schema configured in **Settings** â†’ **Thread fields**.

Thread fields can be nested and be either a boolean, text or a string enum.

Thread fields can be required. When they are required, their value must be set in order for the thread to be marked as done.

For interacting with thread fields via the API, every field has a `key` defined in it's schema. Keys make it possible to quickly refer to a thread field without having to know it's ID in the schema. For example if you have a field called "Product Area" the key you might choose for the key to be `product_area`.
